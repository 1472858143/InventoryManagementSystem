# Agent 协同工作流

## 适用范围

本文档适用于 `Document/3-项目开发/迭代优化/` 下的所有持续迭代任务。目标是让 Codex、Claude Code CLI 等 Agent 在同一项目中按统一方式记录任务、分析影响、执行修改和交付验证结果。

## 工作原则

- 先记录任务，再修改代码。
- 先分析影响，再调整数据库或业务逻辑。
- 先在任务目录中形成变更脚本，再决定是否更新正式 SQL。
- 不修改已验收正式文档正文，除非用户明确要求。
- 不删除已有正式文档。

## 标准流程

1. **登记问题**
   - 在 `00-问题清单.md` 增加问题条目。
   - 标明问题来源、影响范围、优先级和当前状态。

2. **创建任务目录**
   - 复制 `tasks/TASK_TEMPLATE/`。
   - 将复制后的目录改名为 `YYYY-MM-DD-任务主题`。
   - 保留模板文件，不删除影响分析文件。

3. **补齐任务上下文**
   - 在任务 `README.md` 中写清目标、范围和非目标。
   - 在 `AGENT_PROMPT.md` 中写清 Agent 执行指令。
   - 在 `设计方案.md` 中记录方案和取舍。

4. **记录影响**
   - 数据库影响写入 `数据库变更.sql`。
   - 前端影响写入 `前端影响.md`。
   - 后端影响写入 `后端影响.md`。
   - 测试结果写入 `测试记录.md`。
   - 实际过程写入 `开发记录.md`。

5. **执行与验证**
   - 每次修改保持范围小而明确。
   - 运行与任务相关的验证命令。
   - 验证失败时先记录原因，不伪造通过状态。

6. **验收与归档**
   - 用户验收后，将任务目录移动到 `archive/`。
   - 需要更新正式文档时，在验收后单独执行。
   - 未验收任务不得写入正式项目书结论。

## 多 Agent 分工建议

- 文档 Agent：维护任务说明、影响分析和归档记录。
- 后端 Agent：处理 Java、Controller、Service、Mapper、测试。
- 前端 Agent：处理 Vue、路由、API 封装、页面交互。
- 数据库 Agent：提出 SQL 变更草案和迁移风险。
- 验证 Agent：运行测试、构建和检查命令。

## 禁止事项

- 禁止直接覆盖 `sql/market.sql`。
- 禁止把未验证方案写成已完成事实。
- 禁止将多个无关优化合并成一个任务。
- 禁止绕过任务目录直接修改正式项目书。
- 禁止删除需求分析、系统设计和项目总览中的现有正式文档。
