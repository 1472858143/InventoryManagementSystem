## 决策记录

- **2026-05-10：修正 IT-001 双数量为"数量+陈列状态"模型**
  - 用户论证后认定 IT-001 的 `shelf_quantity` 精确数量字段不符合真实超市运营（货架数量随顾客购买实时变化，无法精确追踪）。
  - 改为 `quantity`（总库存数量精确）+ `shelf_status` 枚举（未上架/缺货/较少/充足，理货员手动标记）。
  - 这部分回退了 IT-001 的 `shelf_quantity`，但保留了双层流转的设计思想。

- **2026-05-10：信息架构按"静态主数据 vs 动态状态+流转"分组**
  - 否决了初始提的"商品流转 / 库存运营"分法（用户指出职责交叉）。
  - 改为：商品管理 = 静态主数据（商品库 + 分类）；库存管理 = 动态状态 + 所有流转动作（工作台 + 总览 + 入库 + 上架 + 出库 + 盘点）。
  - 用户管理类下沉到"系统"分组最后。

- **2026-05-10：阶段划分用场景纵切法**
  - 否决了"分层推进"和"两阶段重击"两个备选。
  - 选定 P1 数据迁移 → P2 IA+视觉基线 → P3 双层流转场景 → P4 BI 仪表盘 → P5 首页+收尾。
  - 每个 Phase 完成后产出可独立演示的中间状态。

- **2026-05-10：视觉风格选定 A 浅色靛蓝 + Lucide**
  - 否决了 B 柔和深色和 C 暖中性色。
  - 主色 `#6366F1`，Lucide 线性图标统一替换 ant-design 风格。
