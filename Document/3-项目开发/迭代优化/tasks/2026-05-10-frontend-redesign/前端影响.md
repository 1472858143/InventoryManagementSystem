# 前端影响

## 是否涉及前端

- 结论：是，本任务以前端重设计为主轴。
- 影响页面：
  - 重构（新视觉骨架）：HomeView、UserView、ProductView、CategoryView、StockView、InboundView、OutboundView、StockcheckView、ReportView、SystemView
  - 新增：StockWorkspaceView（库存管理工作台）、ShelfRestockView（上架补货）、ReportDashboard 多 Tab 子页
  - 调整：LoginView 视觉对齐新令牌即可，结构不变
- 影响组件：
  - AdminLayout（侧边栏分组、Lucide 图标接入、视觉令牌应用）
  - 新增 PageContainer、StatCard、Workspace、ShelfStatusBadge 等共享组件
- 影响 API 封装：
  - `api/stock.js`：字段名变更（warehouseQuantity + shelfQuantity → quantity + shelfStatus）
  - `api/report.js`：新增多个仪表盘聚合接口封装
  - `api/inbound.js`、`api/outbound.js`：返回的库存联动字段对齐
  - 新增 shelf-restock 操作 API（如 PATCH /stocks/{id}/shelf-status）

## 用户交互变化

- **侧边栏**：从 10 项平铺改为 4 个一级菜单分组（首页 / 商品管理 / 库存管理 / 报表分析 / 系统），二级菜单可展开收起，URL 直达。
- **库存管理入口**：默认进入工作台首屏（指标 + 缺货预警 + 任务入口），不再直接进库存表格。
- **库存总览页**：每行显示总库存数量 + 陈列状态徽章（颜色编码：未上架灰、缺货红、较少橙、充足绿），新增"改陈列状态"快捷下拉。
- **入库单据**：保留现有交互，但表单字段提示更新为"入库后增加总库存数量"，提交后不再涉及"上架数量"。
- **出库单据**：保留现有交互，提交后不再扣减"货架数量"，统一扣减总库存。
- **上架补货**（新页面）：批量列表 + 多选 + 批量改 shelf_status，可按当前状态过滤。
- **库存盘点**：流程化重构为"选范围 → 生成盘点单 → 录入实际数量 → 差异审核 → 调整 quantity"四步骤。
- **报表分析**：从 3 张图改为 4 个 Tab 仪表盘（库存健康度 / 销售分析 / 决策辅助 / 流转明细），每个 Tab 内多图表 + 指标卡 + 可下钻。
- **首页驾驶舱**：4 张统计卡 + 入出库趋势图保留并视觉重设计；新增"今日缺货"卡片和"补货待办"卡片。
- **图标系统**：所有 ant-design 风格图标替换为 Lucide 线性图标。

## 数据契约变化

- **GET /stocks** 响应字段：
  - 删除 `warehouseQuantity`、`shelfQuantity`
  - 新增 `quantity`（int）、`shelfStatus`（string: 未上架/缺货/较少/充足）
- **POST /stocks**、**PUT /stocks/{id}**：
  - 请求字段同上调整
- **新增 PATCH /stocks/{id}/shelf-status**：
  - 请求体：`{ shelfStatus: string }`
  - 响应：更新后的 stock 完整对象
- **新增 POST /stocks/shelf-status/batch**：
  - 请求体：`{ ids: number[], shelfStatus: string }`
  - 响应：成功更新的记录数
- **新增 GET /reports/stock-turnover**：库存周转率
- **新增 GET /reports/restock-suggestions**：补货建议清单
- **新增 GET /reports/slow-moving**：滞销商品列表
- **新增 GET /reports/sales-hotspot**：销售热点商品
- **新增 GET /reports/category-distribution**：分类库存占比
- **库存日志 stock_log**：因 IT-001 已包含 stock_type 字段，不需要前端额外改动；但日志展示页（如有）需对齐新字段语义。

## 兼容性风险

- **字段重命名风险**：若有未发现的旧代码引用 `warehouseQuantity` 或 `shelfQuantity`，会运行时报错。建议 P1 完成后全文搜索这两个字符串确认零残留。
- **HomeView 库存预警逻辑**：当前用 `s.quantity < s.minStock` 判断（来自 IT-001 前的 stock 字段），P1 后该判断仍可工作（quantity 字段重命名后仍存在），但语义变了——现在 quantity 是"总库存"，判断 quantity < minStock 仍然合理。
- **仪表盘新接口性能风险**：库存周转率、滞销识别等聚合查询若数据量大可能慢，建议加合适索引（已在 `数据库变更.sql` 中加 shelf_status 索引）。
- **Lucide 图标包体积**：Lucide 全量包较大，建议按需引入（`lucide-vue-next` 支持 tree-shaking）。
- **设计令牌覆盖范围**：旧组件 scoped style 内的硬编码颜色需在 P2 阶段统一搬到 tokens.css，否则后续主题切换不彻底。
- **Element Plus 主题色覆盖**：靛蓝 `#6366F1` 与 Element Plus 默认 `#409EFF` 蓝色不同，需通过 SCSS 变量覆盖或 CSS 变量重定义 `--el-color-primary`。

## 前端验证

- **构建命令**：
  - `cd frontend && npm install`（首次或新增依赖后）
  - `cd frontend && npm run build`
- **运行命令**：
  - `cd frontend && npm run dev`
- **手动验证路径**：
  - P1：登录后访问原 10 个页面，确认数据可读可写，控制台无字段相关报错。
  - P2：登录后查看侧边栏分组结构正确、视觉风格符合方案 A、所有图标为 Lucide 线性风格。
  - P3：完整跑通双层流转故事（详见 README 验收标准）。
  - P4：访问报表分析，验证 4 个 Tab 数据正确加载、图表交互可用。
  - P5：首页驾驶舱指标准确、商品管理细节对齐、整站视觉无破绽。
- **自动化测试**：
  - 项目无前端单元测试基线，本任务不新增。
  - 验收依赖手动测试 + `npm run build` 编译通过。
