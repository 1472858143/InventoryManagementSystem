# 前端影响

## 是否涉及前端

- 结论：涉及，但第一阶段只做结构设计，不改前端代码。
- 影响页面：`frontend/src/layout/AdminLayout.vue`、`frontend/src/router/index.js`、首页、入库、出库、库存、报表、商品、用户、系统相关页面。
- 影响组件：待设计确认后决定是否抽取当前库存查询、单据查询、统计卡片等公共组件。
- 影响 API 封装：后续可能新增 supplier、customer、purchase、sales、stock-adjustment、system-log 等 API 封装。

## 用户交互变化

拟设计的新菜单结构为：

- 首页
- 进货管理：进货入库、退货出库、进货单据查询、退货单据查询、当前库存查询
- 销售管理：销售出库、客户退货、销售单据查询、客户退货查询、当前库存查询
- 库存管理：商品报损、商品报溢、库存报警、报损报溢查询、当前库存查询
- 统计报表：供应商统计、客户统计、商品采购统计、商品销售统计、报表分析
- 基础资料：供应商管理、商品管理
- 系统管理：角色管理、用户管理、系统日志、密码修改、安全退出

## 数据契约变化

第一阶段不改请求参数和响应字段。后续实现时，预计会新增采购、销售、退货、报损报溢、供应商、客户、系统日志等接口契约。

## 兼容性风险

- 当前多个新菜单都包含“当前库存查询”，需要设计为复用同一页面或公共组件，避免重复实现。
- 原“报表分析”“系统”“商品管理”“库存管理”菜单位置会调整，演示截图和文档需要同步。
- 如果后续引入角色权限，不同角色可见菜单需要和后端权限一致。

## 前端验证

- 构建命令：本阶段不运行前端构建，未修改前端代码。
- 手动验证路径：本阶段只验证文档结构。
- 自动化测试：无。
