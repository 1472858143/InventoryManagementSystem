# 前端影响

## 是否涉及前端

- 结论：涉及，但当前阶段不修改前端代码。
- 影响页面：`frontend/src/views/product/ProductView.vue`
- 影响组件：当前商品页面内的表格、弹窗表单、状态操作按钮。
- 影响 API 封装：当前阶段不修改 `frontend/src/api/product.js`。

## 当前页面现状

- 页面标题为商品管理，说明文字为维护商品基础信息和上下架状态。
- 列表字段包括商品编号、商品名称、分类、单位、进价、售价、状态、创建时间和操作。
- 新增商品通过弹窗表单完成。
- 分类数据在弹窗打开时调用分类接口刷新。
- 上下架状态通过商品状态更新接口完成。
- 异常提示优先使用接口异常中的 `message`。

## 用户交互变化

当前阶段不产生用户交互变化。后续设计阶段再明确页面展示、表单、按钮、提示和状态反馈是否调整。

## 数据契约变化

当前阶段不改变请求参数、响应字段、状态码或错误消息。

## 兼容性风险

- 后续如果调整商品列表字段或展示层级，需要确保仍使用现有后端返回字段。
- 后续如果增加前端筛选或排序，应优先在前端展示层完成，避免误认为后端已支持分页或筛选。
- 商品页面不能处理库存计算或库存修改规则。

- 构建命令：本阶段未运行。
- 手动验证路径：后续进入实现阶段后验证商品管理页面。
- 自动化测试：本阶段不涉及。
