# 前端影响

## 是否涉及前端

- 结论：涉及。
- 影响页面：库存菜单组新增商品报损、商品报溢、报损报溢查询、库存报警。
- 影响组件：可复用 IT-006 的明细编辑、汇总栏、查询条、详情抽屉；必要时扩展以适配 `type` 和可空金额。
- 影响 API 封装：新增 `stockAdjustment` 和 `stockAlarm` 相关 API。

## 用户交互变化

- 商品报损页：选择商品、数量、可选单价、原因，保存后库存减少。
- 商品报溢页：选择商品、数量、可选单价、原因，保存后库存增加。
- 报损报溢查询页：支持类型、单据号、商品、操作人、日期筛选；可查看详情和作废。
- 库存报警页：展示低于下限或高于上限的商品库存。

## 数据契约变化

- 新增创建库存调整单请求：`type`、`operatorId`、`reason`、`items[]`。
- 新增查询库存调整单响应：主表字段、明细字段、分页结构。
- 新增库存报警查询响应：商品、库存数量、上下限和报警类型。

## 兼容性风险

- 库存页面菜单组会变长，需要确保侧边栏滚动正常。
- 明细编辑组件若复用采购 / 销售页面逻辑，要允许单价为空。
- 报溢作废可能因库存不足失败，前端需展示后端错误。

## 前端验证

- 构建命令：`cd frontend && npm run build`
- 手动验证路径：`/stock/loss`、`/stock/overflow`、`/stock/adjustments`、`/stock/alarms`
- 自动化测试：本项目当前未建立前端单测，至少执行构建和页面烟测。
