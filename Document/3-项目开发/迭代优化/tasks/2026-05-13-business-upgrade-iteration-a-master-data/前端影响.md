# 前端影响

## 是否涉及前端

- 结论：涉及。
- 影响目录：`frontend/src/views/master/`（新增）、`frontend/src/router/index.js`、`frontend/src/layout/AdminLayout.vue`、`frontend/src/api/`（新增 supplier.js / customer.js）、商品页相关组件。
- 影响组件：商品表单内嵌分类下拉（新建或复用现有分类选择组件）。

## 用户交互变化

新增"基础资料"菜单组，内含 3 项：

- 供应商管理：列表 / 新增 / 编辑 / 启停用 / 按编码或名称搜索。
- 客户管理：同上结构。
- 商品管理：从原位置迁入；原"商品分类管理"独立菜单下线，分类选择内嵌到商品表单。

旧菜单变化：

- "商品分类管理"独立菜单：下线，路由 redirect 到商品管理或显式 404。
- "商品管理"：菜单位置从原位置迁到"基础资料"组下；路径与组件不动以减少风险。

## 数据契约变化

本任务新增 API：

- `GET /api/suppliers`、`POST /api/suppliers`、`PUT /api/suppliers/{id}`、`PATCH /api/suppliers/{id}/status`
- `GET /api/customers`、`POST /api/customers`、`PUT /api/customers/{id}`、`PATCH /api/customers/{id}/status`

商品 API 与分类 API 沿用，前端调用方式调整为内嵌使用。

## 兼容性风险

- 旧"商品分类管理"路由下线后，若有外部链接 / 收藏夹访问，须 redirect 到商品页或 404 提示。
- 菜单顺序与位置变化，演示截图与项目书材料后续统一同步（验收后再做）。
- 编码唯一约束的前端校验需要与后端一致，避免提交后才被驳回。

## 前端验证

- 构建命令：`cd frontend && npm run build`。
- 手动验证路径：
  - 基础资料菜单组三项是否齐全。
  - 供应商 / 客户 CRUD 与启停用是否跑通。
  - 商品页内嵌分类下拉是否正常加载与选择。
  - 旧分类菜单是否不可见 / 不可达。
- 自动化测试：暂无前端测试覆盖；以手动验证为准。
