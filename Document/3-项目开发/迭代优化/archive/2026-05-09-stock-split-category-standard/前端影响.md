# 前端影响

## 是否涉及前端

- 结论：是，多个页面需要更新
- 影响页面：StockView、ProductView、InboundView（小）、OutboundView（小）、可能新增 CategoryView
- 影响组件：库存表格列、商品新增/编辑 Dialog、补货 Dialog（新增）
- 影响 API 封装：`frontend/src/api/product.js`（已有）、新增 `frontend/src/api/category.js`、`frontend/src/api/stock.js` 补货方法

## 用户交互变化

### StockView（库存管理页）

**变更前**：表格显示 productCode、productName、quantity（单一库存）、minStock、maxStock

**变更后**：
- 表格新增列：仓库库存（warehouse_quantity）、上架库存（shelf_quantity）、总计（两者之和）
- 预警标签逻辑改为基于**上架库存（shelf_quantity）**与 min/max 对比（已确认）：`shelf_quantity < min_stock` → "上架不足"；`shelf_quantity > max_stock` → "上架过多"
- 新增"补货"操作按钮，点击弹出补货 Dialog
- 补货 Dialog 字段：选择商品（已在行中确定）、补货数量（el-input-number，min=1，max=当前 warehouse_quantity）、操作人（自动填充当前登录用户）
- 补货 Dialog 展示当前仓库库存和上架库存，帮助用户判断补货数量

### ProductView（商品管理页）

**变更前**：新增商品 Dialog 中分类为 el-input 文本输入

**变更后**：
- 分类改为 el-select 下拉，选项来自 `/categories` 接口（启用状态），支持 filterable
- 新增"单位"字段：el-select 提供常用单位（件、箱、kg、g、L、mL、瓶、袋）+ 允许自定义输入（el-autocomplete 或 el-select 带 allow-create）
- 表格列：category 列改为显示 categoryName（从后端 JOIN 返回），unit 列新增

### InboundView（入库管理页）

**变更前**：入库成功后库存增加（用户不感知是哪种库存）

**变更后**：
- 页面说明文字更新为"入库操作将增加仓库库存，不直接影响上架库存"
- 提交成功后的 ElMessage 提示更新为"入库成功，已增加仓库库存"
- 无结构性改动

### OutboundView（出库/购买页）

**变更前**：出库从总库存扣减

**变更后**：
- 页面说明文字更新为"出库操作将减少上架库存，仓库库存不受影响"
- 若后端返回"上架库存不足"错误，前端显示具体提示"上架库存不足，请先在库存管理页面进行补货操作"
- 无结构性改动

### CategoryView（分类管理页，已确认需要）

- 路由：`/categories`，侧边栏菜单新增"商品分类"入口（位于"商品管理"下方）
- 新建 `frontend/src/views/category/CategoryView.vue`
- 新建 `frontend/src/api/category.js`
- 页面功能：
  - 表格显示全部分类：ID、分类名称、状态标签（启用绿/禁用灰）、创建时间、操作列
  - 操作列：启用/禁用切换按钮
  - 页面右上角"新增分类"按钮，点击弹出 Dialog
  - 新增 Dialog：分类名称输入（required，blur 校验），提交调用 POST `/categories`
  - 禁用后分类不出现在商品新增/编辑的下拉列表中（前端过滤 status=1）

## 数据契约变化

| 字段 | 变更说明 |
| --- | --- |
| `stock.quantity` | 废弃，改为 `warehouseQuantity` + `shelfQuantity` |
| `product.category` | 废弃，改为 `categoryId` + `categoryName` |
| `product.unit` | 新增，字符串 |
| 补货接口 POST `/stocks/{id}/restock` | 新增，请求体 `{quantity, operator}` |
| 分类列表接口 GET `/categories` | 新增，返回 `[{id, categoryName}]` |

## 兼容性风险

- 前端 `StockView.vue` 中所有 `row.quantity` 引用需改为 `row.warehouseQuantity + row.shelfQuantity` 或分别显示
- 前端 `ProductView.vue` 中 `row.category` 显示需改为 `row.categoryName`
- 入库/出库 Dialog 中 products 数据源无变化（productId、productName 仍可用），但商品列表的响应结构变化需确认兼容

## 前端验证

- 构建命令：`cd frontend && npm run build`（确认无编译错误）
- 手动验证路径：
  1. 库存页 → 确认三列库存数字正确；点击补货 → 填写数量 → 提交 → 数字变化正确
  2. 商品页 → 新增商品 → 分类为下拉 → 单位可选/输入 → 保存成功
  3. 入库页 → 正常入库 → 返回库存页确认仓库库存增加
  4. 出库页 → 正常出库 → 返回库存页确认上架库存减少
  5. 出库页 → 超出上架库存 → 显示"上架库存不足"提示
- 自动化测试：当前项目无自动化 UI 测试，以手动验证为准
