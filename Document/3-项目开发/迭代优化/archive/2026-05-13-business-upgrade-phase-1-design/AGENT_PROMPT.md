# Agent 执行提示

## 任务目标

请先阅读本任务目录中的全部文件，再开始分析或修改。执行目标以 `README.md` 中的目标和验收标准为准。本任务是业务升级第一阶段，只做设计和影响分析，不直接进入代码实现。

## 执行约束

- 不直接修改正式项目书、需求分析和系统设计文档正文。
- 不直接覆盖 `sql/market.sql` 或 `Document/3-项目开发/market.sql`。
- 数据库调整先写入本任务目录的 `数据库变更.sql`。
- 修改前检查 Git 状态，避免覆盖用户未提交内容。
- 每次变更保持范围小而明确。
- 未验证内容必须标明为待确认或待验证。

## 必读文件

1. 根目录 `AGENTS.md`
2. `Document/3-项目开发/迭代优化/AGENT_WORKFLOW.md`
3. `Document/业务升级构想工作量与改造影响分析.md`
4. 当前任务 `README.md`
5. 当前任务影响分析文件

## 设计重点

- 模块边界：首页、进货管理、销售管理、库存管理、统计报表、基础资料、系统管理。
- 数据库草案：供应商、客户、采购/销售单据、退货、报损报溢、系统日志、权限。
- 库存统一变更：采购入库、采购退货、销售出库、客户退货、报损、报溢、盘点调整的变更类型和来源单据。
- 前端菜单：旧菜单如何重命名、迁移、复用或移除。
- 后续迭代拆分：哪些内容先做，哪些内容后做。

## 输出要求

- 汇报实际修改文件。
- 汇报验证命令和结果。
- 汇报未完成事项和风险。
- 不把未验证内容表述为已完成。
