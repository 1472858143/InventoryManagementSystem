# 前端影响

## 是否涉及前端

- 结论：涉及，本任务以前端为主。
- 影响页面：`frontend/src/views/product/ProductView.vue`（重构）。
- 影响组件：新增 `ProductCard.vue`、`ProductDetailDrawer.vue`、`ProductFilterBar.vue`；新增 `frontend/src/utils/pinyinSort.js`。
- 影响 API 封装：不修改 `frontend/src/api/product.js`，继续使用 `getProducts / createProduct / updateProductStatus` 三个方法；继续使用 `getEnabledCategories`。
- 影响依赖：`frontend/package.json` 与 lock 文件新增 `pinyin-pro`。

## 页面结构变化

### 前

`ProductView.vue` 单文件包含：

- 顶部页头（标题 + 刷新 + 新增按钮）。
- `el-card` 包裹 `el-table`，列：商品编号 / 名称 / 分类 / 单位 / 进价 / 售价 / 状态 / 创建时间 / 操作。
- 新增弹窗 `el-dialog` 表单：商品编号 / 名称 / 分类 / 单位 / 进价 / 售价。
- 状态切换：表格行内「上架 / 下架」link 按钮。

### 后

`ProductView.vue` 重构为容器，内部组合：

- 顶部页头（标题 + 简短描述）。
- `ProductFilterBar` 工具栏：搜索输入框 + 排序下拉 + 筛选展开按钮 + 新增商品按钮。展开后显示售价区间、进价区间、分类多选、状态单选、重置按钮。
- 卡片网格区：CSS Grid，渲染 `ProductCard` 数组，data 来源为 `filteredAndSortedProducts` computed。
- `ProductDetailDrawer` 抽屉：v-model:visible 控制；接收当前选中商品；内含信息分区、状态管理分区、扩展槽位（注释预留）。
- 新增弹窗 `el-dialog` 表单：去掉「商品编号」字段，其他保留。

## 用户交互变化

| 场景 | 前 | 后 |
| --- | --- | --- |
| 浏览商品 | 表格滚动 | 卡片网格扫视，状态色彩快速识别 |
| 查看详情 | 全部字段挤在表格列 | 点击卡片打开右侧抽屉看分区详情 |
| 切换状态 | 表格行内点 link | 在详情抽屉的状态管理分区切换 |
| 新增商品 | 必填编号 | 编号字段移除，无需手填 |
| 检索 | 无 | 搜索框输入名称/分类即时过滤 |
| 多条件检索 | 无 | 筛选面板组合售价/进价/分类/状态 |
| 排序 | 无 | 排序下拉切换 5 种排序键，默认销量降序 |

## 数据契约变化

- `getProducts` 返回新增字段 `salesCount: number`（int，可能为 0）。
- `createProduct` 请求 body 不再包含 `productCode` 字段（后端可选，缺省则自动生成）。
- 其他字段、响应结构、错误码不变。

## 视觉与设计 token

- 复用 `frontend/src/styles/tokens.css` 已有变量：
  - 上架状态边框：`var(--color-success)` (#10B981)。
  - 下架状态边框与文字：`var(--color-text-secondary)` (#94A3B8)。
  - 卡片背景：`var(--color-white)`；阴影：`var(--shadow-card)`；圆角：`var(--radius-md)`。
  - 主色按钮、tag：沿用 `--color-primary` (#6366F1)。
- 不引入新的颜色变量；如确需扩展，统一加入 tokens.css，不在组件内硬编码 hex。

## 兼容性风险

- pinyin-pro 体积约 ~30KB gzipped，构建产物会增加；但比备选 `pinyin` 小且 ESM 友好。
- 卡片网格在窄屏（<480px）会自动单列展示，需要在最小宽度 320px 设备验证。
- 详情抽屉打开时屏幕宽度小于 600px 应自适应为 90vw，避免遮挡过多内容。
- 全前端筛选/排序在数据量超过 1000 条时性能下降；本毕设场景预期 < 200 条，不需要分页。
- 商品列表若有空 `categoryName` 或空 `productName`，搜索/排序需做空值兜底（`?? ''`）。

## 前端验证

- 类型检查：项目暂无 ts，依赖 ESLint。

- 自动化测试：本任务不引入前端单测（项目无前端测试基础设施）。
