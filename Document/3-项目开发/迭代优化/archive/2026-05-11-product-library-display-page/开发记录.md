## 决策记录

- 本任务范围限定为商品库展示页面优化。
- 设计阶段（2026-05-11）确认 4 个关键设计决策：
  1. 商品编号策略：前端表单删除编号字段 + 后端在 createProduct 中自动生成（格式 `P` + `yyyyMMddHHmmss` + 4 位随机数字）。DB 列与唯一索引保留。
  2. 销量来源：聚合 `stock_log` 表 `change_type = 'OUTBOUND'` 的 `change_quantity`，在 `findAllWithCategory` 查询里 LEFT JOIN 子查询，附带为 `salesCount` 字段返回。
  3. 详情容器：使用 `el-drawer` 右侧抽屉，宽度 520px，分区结构便于后续扩展折扣等模块。
  4. 任务目录：在现有 `2026-05-11-product-library-display-page` 原地升级，不另建目录。
- 分支使用 `feature/product-library-display-page`。
- `AGENTS.md` 的状态更新纳入本次迭代改动一起提交。
- 实施分四个阶段（P1 后端微调 → P2 前端基础组件 → P3 ProductView 重构 → P4 验证），每阶段独立提交，避免大爆炸合并。

## 问题与处理

- 设计提示中提到 `ProductMapper.xml`，但当前仓库实际为注解式 `ProductMapper.java`，因此销量聚合落在 `ProductMapper.java` 的 `@Select` SQL 中。
- P4 浏览器走查时发现详情抽屉自定义头部缺少显式关闭按钮，已补充右上角关闭图标按钮并复查通过。
- P4 控制台检查曾出现 Element Plus `el-radio` 的 label-as-value 弃用警告，已将 `ProductFilterBar` 的 `el-radio-button` 改为 `value` API 并重新构建。
