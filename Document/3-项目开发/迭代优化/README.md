# 迭代优化

## 当前项目状态

项目已从一次性毕业设计开发进入持续迭代优化阶段。当前正式项目书、需求分析、系统设计和正式 SQL 视为已验收基线，后续优化必须通过任务化方式记录、验证和归档。

## 文档体系

- `README.md`：迭代优化目录导航和当前规则入口。
- `AGENT_WORKFLOW.md`：多 Agent 协同开发流程。
- `00-问题清单.md`：待分析、待拆分、待排期的问题池。
- `tasks/`：进行中的迭代任务目录。
- `tasks/TASK_TEMPLATE/`：新任务复制使用的统一模板。
- `archive/`：已完成、已暂停或已废弃任务的归档目录。

## 文档优先级

1. 用户在当前任务中的明确指令。
2. 根目录 `AGENTS.md`。
3. 本目录 `AGENT_WORKFLOW.md`。
4. 具体任务目录中的任务文档。
5. 已验收正式文档：项目书、需求分析、系统设计、正式 SQL。

## 任务工作流

1. 在 `00-问题清单.md` 登记问题或优化点。
2. 从 `tasks/TASK_TEMPLATE/` 复制新任务目录。
3. 在任务目录内补齐目标、范围、影响分析、验证方式。
4. Agent 按任务文档执行，不直接修改正式基线文档。
5. 任务完成后记录测试结果和开发记录。
6. 用户验收后，再决定是否同步更新正式项目书、系统设计文档或 SQL。

## Agent 协同规则

- 一个任务目录对应一个优化主题。
- 不同 Agent 同时工作时，应优先选择互不重叠的任务目录。
- 数据库变更必须写入任务目录中的 `数据库变更.sql`，不得直接覆盖正式 SQL。
- 对正式文档的更新必须在任务验收后统一执行。
- 所有未验证判断必须写明验证缺口。

## 新任务命名建议

使用日期和主题命名：

```text
tasks/2026-05-09-product-status-optimization/
tasks/2026-05-09-stock-flow-refactor/
```
