# 系统实施与开发操作步骤说明

本文基于既定的系统架构设计，给出从项目启动到系统部署的完整实施步骤，用于指导系统的实际开发与落地实现。

---

## 一、整体实施顺序说明

为保证系统开发过程的稳定性与可控性，项目实施遵循“先核心、后扩展，先后端、后前端”的原则，整体推荐实施顺序如下：

1. 数据库设计  
2. 后端项目骨架搭建  
3. 核心领域模块（库存）实现  
4. 其他业务模块实现  
5. 前端项目开发  
6. 前后端联调  
7. 云服务器部署与上线  

---

## 二、数据库设计阶段

### 2.1 数据库设计优先级

数据库设计是系统开发的第一步，其结构将直接影响后端实体类、业务逻辑与接口设计。

推荐按照以下优先级进行设计：

#### （1）基础核心表
- 用户表（user）
- 角色表 / 用户角色关联表（role / user_role）
- 商品表（product）
- 库存表（stock）

#### （2）核心业务表
- 入库单表（inbound_order）
- 出库单表（outbound_order）
- 库存盘点表（stock_check）

#### （3）日志与扩展表
- 库存变更流水表（stock_log）
- 系统操作日志表（operation_log）

### 2.2 数据库设计流程

1. 根据业务需求绘制 E-R 图  
2. 明确实体之间的关系（一对一、一对多、多对多）  
3. 设计数据库表结构（字段、类型、主键、外键）  
4. 编写数据库建表 SQL 脚本  

---

## 三、后端项目骨架搭建

### 3.1 项目形式选择

后端采用 **单体 Spring Boot 项目**，通过包结构实现模块化管理。

说明：
- 当前阶段不拆分为多个后端项目
- 通过业务模块实现逻辑隔离
- 降低开发、部署与维护复杂度

### 3.2 后端基础工作

在该阶段仅完成基础搭建，不编写具体业务逻辑，主要包括：

- 创建 Spring Boot 项目
- 引入基础依赖（Web、MyBatis/JPA、MySQL Driver 等）
- 搭建项目包结构（controller / service / domain / repository）
- 配置全局异常处理
- 设计统一响应返回结构
- 配置数据库连接与环境配置文件

---

## 四、核心领域模块优先实现（库存模块）

### 4.1 核心领域优先实现原因

库存模块是系统的核心业务领域，入库、出库、盘点等功能均依赖库存模块完成库存数量变更，因此需要优先实现。

### 4.2 库存模块实现步骤

1. 创建库存相关数据库表（stock、stock_log）  
2. 编写库存实体类（StockEntity）  
3. 编写库存数据访问层（StockRepository）  
4. 实现库存领域服务（StockDomainService），作为库存变更的唯一入口  
5. 实现库存业务服务层（StockService）  
6. 提供库存相关接口（StockController）  

库存领域服务负责：
- 校验库存变更合法性
- 执行库存增减操作
- 记录库存变更流水
- 触发库存预警规则

---

## 五、其他业务模块实现顺序

在库存模块稳定后，按依赖关系逐步实现其他业务模块，推荐顺序如下：

1. 商品管理模块（product）  
2. 入库管理模块（inbound）  
3. 出库管理模块（outbound）  
4. 库存盘点模块（stockcheck）  
5. 采购与供应商模块（purchase）  
6. 报表统计模块（report，仅提供查询功能）  

说明：
- 涉及库存变更的模块必须调用库存领域服务
- 禁止直接操作库存表

---

## 六、前端项目开发阶段

### 6.1 前端项目职责

前端项目负责用户界面展示与交互，不包含业务逻辑处理，主要包括：

- 页面布局与导航结构
- 用户登录与权限控制
- 业务页面展示
- 调用后端接口并展示数据

### 6.2 前端项目基础结构建议

```text
src
├─ api          // 后端接口封装
├─ views        // 页面组件
├─ router       // 路由配置
├─ store        // 状态管理
├─ utils        // 请求与工具封装
└─ layout       // 页面整体布局
```

前端通过统一的 API 模块调用后端接口，避免在页面组件中直接拼接请求地址，从而提高代码的可维护性与可读性。

---

## 七、前后端通信方式说明

### 7.1 通信协议与数据格式

- 通信协议：HTTP / HTTPS  
- 数据格式：JSON  
- 接口风格：RESTful  

---

### 7.2 接口交互示例

**请求示例：**

```http
POST /api/inbound/create
Content-Type: application/json
Authorization: Bearer token
```

**请求体示例**

```json
{
  "productId": 1,
  "quantity": 100
}
```

**返回结果示例**
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

---
### 7.3跨域问题处理方式
- 前端统一通过 Nginx 访问后端接口
- 前端请求路径统一以 /api 作为前缀
- 由 Nginx 将请求反向代理至 Spring Boot 后端服务
- 后端无需额外配置 CORS 跨域策略

---

## 八、云服务器部署说明（华为云 ECS）
### 8.1 云服务器部署内容
在华为云 ECS 上需要部署的组件包括：
- Nginx
- Spring Boot 后端服务（Jar 包）
- MySQL 数据库（或云数据库服务）
可选组件：
- Redis
- 对象存储服务(图片)

---

### 8.2 部署关系说明
客户端（浏览器）
  ↓
Nginx（ECS）
  ├─ Vue 静态资源
  └─ /api 请求 → Spring Boot
                    ↓
                  MySQL
说明：
- 前端静态资源不部署在 Spring Boot 服务中
- Nginx 作为系统的统一访问入口

---

## 九、项目拆分方式说明

### 9.1 是否拆分为多个项目

本系统采用如下项目拆分方式：

- 后端：一个 Spring Boot 项目（模块化单体架构）
- 前端：一个 Vue 项目
- 数据库：一个数据库实例
- 
当前阶段不建议拆分为多个后端项目或微服务，以避免增加系统开发、部署与维护的复杂度。
