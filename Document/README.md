# Document 文档目录

本目录只用于存放项目文档，当前已按项目阶段整理，并把可合并的碎片文档汇总为少量总文档。

## 目录说明

- `0-项目总览/`：项目书、需求基线、总体说明等入口材料。
- `1-需求分析/`：需求分析相关文档。
- `2-系统设计/`：系统架构、总体设计、API 接口设计、数据库设计汇总、详细模块设计汇总。
- `3-项目开发/`：开发规划与步骤汇总、`market.sql` 开发阶段副本等材料。
- `4-开发记录/`：后端模块开发记录汇总、前端开发规划。
- `5-过程材料/`：中期报告、毕业设计文书流程等过程性材料。

## 合并记录

- `2-系统设计/数据库设计/` 下的数据库说明、AI 参考、各表设计文档已合并为 `2-系统设计/数据库设计汇总.md`。
- `2-系统设计/详细模块设计/` 下的模块设计文档已合并为 `2-系统设计/详细模块设计汇总.md`。
- `3-项目开发/开发规划/` 和 `3-项目开发/开发步骤/` 下的文档已合并为 `3-项目开发/项目开发规划与步骤汇总.md`。
- `4-开发记录/backend/` 下的后端模块实现、接口测试、注意事项、设计参考等文档已合并为 `4-开发记录/后端模块开发记录汇总.md`。
- `4-开发记录/frontend/前端开发规划.md` 已上移为 `4-开发记录/前端开发规划.md`。
- `5-过程材料/中期报告/超市库存管理系统中期报告.docx` 已上移为 `5-过程材料/超市库存管理系统中期报告.docx`。
- 空白文本文件和 URL 编码空目录没有有效正文内容，已清理。

## 使用建议

- 需要看完整设计时，优先阅读 `2-系统设计/数据库设计汇总.md` 和 `2-系统设计/详细模块设计汇总.md`。
- 需要查看前后端接口契约时，优先阅读 `2-系统设计/API接口设计文档.md`。
- 需要追踪开发过程时，优先阅读 `3-项目开发/项目开发规划与步骤汇总.md` 和 `4-开发记录/后端模块开发记录汇总.md`。
- 需要执行正式数据库脚本时，优先使用项目根目录 `sql/market.sql`；`3-项目开发/market.sql` 作为文档目录内的同步副本。
