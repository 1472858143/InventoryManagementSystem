# AGENTS.md

## 基本规则

- 始终使用简体中文回复。
- 修改前先确认当前 Git 状态，避免覆盖用户未提交内容。
- 不直接修改已验收的正式文档正文，除非用户明确要求。
- 不直接覆盖 `sql/market.sql` 或 `Document/3-项目开发/market.sql`。
- 涉及数据库结构调整时，先在迭代任务中新增变更脚本和说明，再由用户确认是否进入正式 SQL。

## 文档优先级

1. 用户当前对话中的明确指令。
2. 根目录 `AGENTS.md`。
3. `Document/3-项目开发/迭代优化/AGENT_WORKFLOW.md`。
4. 具体任务目录中的 `AGENT_PROMPT.md` 和任务文档。
5. 已验收正式文档：项目书、需求分析、系统设计、正式 SQL。

## 项目状态

- 当前项目后端主体模块与已登记的迭代优化任务已完成并归档。
- 当前阶段进入前端展示页面优化与前后端联调优化阶段。
- `Document/0-项目总览/项目书.md` 作为已验收基线使用。
- 每次迭代优化先在 `Document/3-项目开发/迭代优化/tasks/` 下建立任务记录。
- 迭代完成并验收后，再统一评估是否更新正式项目书、系统设计文档和正式 SQL。

## Agent 协同规则

- 每个 Agent 只处理一个明确任务，不在同一轮中混入无关重构。
- 开始任务前阅读任务目录中的 `README.md`、`AGENT_PROMPT.md` 和影响分析文件。
- 代码、数据库、前端、后端、测试影响必须分别记录。
- 不能生成虚假开发内容；未验证的内容必须标明为待验证。
- 完成后汇报修改文件、验证命令、验证结果和未完成风险。

## 提交流程

- 提交前必须运行与任务相关的验证命令。
- 提交信息使用简体中文，格式建议为：`type: 简短说明`。
- 不将临时输出、构建产物、Agent 草稿缓存提交到正式代码库。
